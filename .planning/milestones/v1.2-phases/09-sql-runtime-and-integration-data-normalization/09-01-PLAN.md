    Create `core/sql/normalization.py` with these concrete function signatures:
    - `def serialize_sql_value(value: Any) -> Any`
    - `def infer_field_type(value: Any) -> str`
    - `def build_sql_fields(columns: Sequence[str], rows: Sequence[Mapping[str, Any]], *, db_type_hints: Mapping[str, str] | None = None) -> list[dict[str, Any]]`
    - `def build_normalized_sql_response(*, rows: Sequence[Mapping[str, Any]], fields: Sequence[Mapping[str, Any]], row_count: int, duration_ms: int, truncated: bool, statement_count: int, statement_types: Sequence[str], is_high_risk: bool, risk_reasons: Sequence[str], timeout_seconds: float, max_rows: int) -> dict[str, Any]`

    Implement canonical field-type taxonomy as exact string values:
    `integer`, `float`, `decimal`, `string`, `boolean`, `date`, `time`, `datetime`, `bytes`, `unknown`.

    Implement deterministic serialization rules:
    - keep JSON-native `str`, `int`, `float`, `bool`, `None` unchanged;
    - serialize `Decimal` to exact string via `str(value)`;
    - serialize `datetime` to ISO-8601 string (`aware` values converted to UTC first);
    - serialize `date` and `time` to `isoformat()` strings;
    - serialize `bytes`, `bytearray`, and `memoryview` to base64 ASCII strings;
    - fallback for unrecognized objects is `str(value)`.

    `build_normalized_sql_response` must output canonical keys and compatibility aliases:
    - canonical: `rows`, `fields`, `row_count`, `duration_ms`, `truncated`, `statement_count`, `statement_types`, `is_high_risk`, `risk_reasons`, `timeout_seconds`, `max_rows`
    - aliases: `columns` (ordered field names), `execution_ms` (same integer as `duration_ms`).

    Export the four functions from `core/sql/__init__.py`.
  </action>
