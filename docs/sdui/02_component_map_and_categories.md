# Glancier 声明式 UI (Schema-Driven UI) 组件地图与分类字典

本文档定义了 Glancier 可视化渲染引擎所支持的基础构建块（Building Blocks）。所有业务卡片级视图必须且只能由以下这五大极简正交组件类别中任意嵌套声明组合而成。

---

## 类别一：Layouts (布局与结构容器)

**核心职责**：自身并不直接显示业务数据内容，专门负责界定空间划分、子元素的排列方向、内容基线对齐方式以及元素间的间距 (Gap)。它是保证卡片 UI “像乐高积木一样拼装”且永远不会相互挤压错位的底层规则。

### 1. `Container` (垂直块状容器)
**功能描述**：
系统默认的垂直流式布局基板。它负责将内部嵌套的所有子组件 (Items) 自上而下流式排列。常作为一张独立 Widget 卡片的根节点，或在卡片内部需要垂直划分为几个显著模块区块时充当分割区间。

### 2. `ColumnSet` (水平栏栅集)
**功能描述**：
用于将屏幕横向空间切分为多个纵列，是处理左右并排展示结构（如：左侧为标题，右侧为徽标状态；或横向均布三个关键数值单元）的核心容器。`ColumnSet` 只能直接包含 `Column` 作为其直系子代。

### 3. `Column` (单列纵栅)
**功能描述**：
仅能存在于 `ColumnSet` 之内的子列元素。它定义了当前列的宽度伸缩规则（由内容撑开自动排版、固定数字权重比例、或占满剩余空间）。在它内部，又可以像一个微型 `Container` 一样自上而下再次排列内容。

---

## 类别二：Containers (数据容器)

**核心职责**：处理数组数据的迭代渲染，支持过滤、排序、分页等数据操作。与 Layouts 不同，Containers 直接与数据源绑定并负责数据处理逻辑。

### 1. `List` (列表数据容器)
**功能描述**：
用于渲染数组类型的数据源。通过 `data_source` 指定数据路径，`item_alias` 定义当前项别名，`render` 配置每个列表项的渲染模板。支持以下能力：
- **布局模式**：`col`（垂直堆叠）或 `grid`（网格布局，可配置列数）
- **数据过滤**：通过 `filter` 表达式筛选数据（如 `”key_item.active == true”`）
- **数据排序**：通过 `sort_by` 指定排序字段，`sort_order` 指定升序/降序
- **数量限制**：通过 `limit` 限制显示条目数
- **分页支持**：通过 `pagination` 和 `page_size` 启用分页

**配置示例**：
```yaml
type: “List”
data_source: “keys_list”
item_alias: “key_item”
layout: “grid”
columns: 2
filter: “key_item.active == true”
sort_by: “key_item.usage”
sort_order: “desc”
limit: 10
pagination: true
page_size: 5
render:
  - type: “TextBlock”
    text: “{key_item.name}”
  - type: “Progress”
    usage: “{key_item.usage}”
    limit: “{key_item.limit}”
```

## 类别三：Elements (原子呈现图元)

**核心职责**：界面中最底层的视觉表达，也是真正在屏幕上画出并承载业务数据的唯一单元。它们本身不具备复杂的布局排版能力，通常只能跟随父容器排列。

### 1. `TextBlock` (万能文本块)
**功能描述**：
负责所有基于字符的信息传达。这是所有元素中最通用的一款组件，从微小的灰色注释、带颜色的上下趋势符号（▲/▼），直到巨大的强调数字（Hero Metric），全部由它通过改变字号、颜色和字重参数来呈现，不再保留独立的特定业务标题组件。

**补充能力**：
- **多行截断**：可通过 `maxLines`（或 `max_lines`）启用 `n` 行截断，避免长文本挤压卡片布局。

### 2. `FactSet` (键值信息表)
**功能描述**：
一种高度规范化呈现的专门组件，用以对齐展现典型的多行 `Label: Value` 摘要资料（例如：账户创建时间、计费周期结束日、云服务区域、当前方案级别等）。

### 3. `Image` (图片与图形)
**功能描述**：
通用图形渲染块。支持根据传入的统一资源定位符（网址或集成方预置的 Icon 标识）进行渲染，自动处理默认回退，并可声明是否将图片裁剪为应用内标准圆形头像尺寸。

### 4. `Badge` (语义状态徽标)
**功能描述**：
带有固化底边底色块、预设圆角及特定内边距样式的短文本状态标牌。虽然其职能看似能用 `TextBlock` 替代，但为了在业务上更鲜明地凸显离散的状态结（如标记资源为 `Active`, `Failed`, `Warning`），故提取为独立语义元素并统一了状态调色板规范。

---

## 类别四：Visualizations (数据可视化面板)

**核心职责**：专门处理业务数据图形化、图表化的高级呈现方式。它们通常不是单纯表现文字，而是以图形比例形式帮助用户快速获取相对量感度量信息。由于其复杂性较高，此类图元应当在开发中坚持极简原则，只有确实没有其他替代呈现方案时才进行扩展。

### 1. `Progress` (进度消耗度量)
**功能描述**：
体现业务资源占用占比、消耗情况或配额达标进度的图形度量仪。支持声明为横向轨道消耗条 (`bar`) 或环形状的圆圈图 (`ring`)，并能够基于当前百分占比通过预设阀门自动联动警告色彩填充逻辑。

---

## 类别五：Actions (交互与行为钩子)

**核心职责**：在纯粹用于“看”的视觉展示界面提供安全、受限且样式统一的操作下发点。将用户触发的外向性事件标准化并加以封装约束。

### 1. `ActionSet` (操作集束栏)
**功能描述**：
一个专门用于水平居齐陈列并包裹动作按钮的底坐容器。它约束了多个可操作项放置在一起时的内部安全间距与对齐排版。

### 2. `Action.OpenUrl` (打开链接跳转)
**功能描述**：
用于在用户点击时明确唤起默认浏览器访问外部网址 URI，或者在特定场景下驱动本机应用系统内的通用路由跳转页。它是链接功能的按钮化实体封装。

### 3. `Action.Copy` (文本复制钩子)
**功能描述**：
提供轻量级的静默剪贴板交互支持。用户点击此动作对应按钮后，将把关联预先组装好的特定文本模版（比如包含 API 密钥片段的字符串信息或 ID）快速下落至操作系统的剪贴板通道。
